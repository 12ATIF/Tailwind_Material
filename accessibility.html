1 Screen Reader Only (sr-only)

Menyembunyikan elemen secara visual tapi tetap terbaca oleh screen reader.

Contoh: Icon button dengan label

Kode:

<button aria-label="Delete">
  <svg>...</svg>
  <span class="sr-only">Hapus item</span>
</button>

üí° Use Case: Icon-only buttons, skip links, loading states, form instructions

2 Not Screen Reader Only (not-sr-only)

Membalik efek sr-only, membuat elemen terlihat kembali.

Contoh: Skip link yang muncul saat focus

Skip to main content

(Tab ke sini untuk melihat skip link)

<a href="#main" class="sr-only focus:not-sr-only
   focus:absolute focus:top-4 focus:left-4 
   focus:px-4 focus:py-2 focus:bg-white focus:text-black">
  Skip to main content
</a>

3 Forced Color Adjust

Mengontrol apakah elemen mengikuti forced colors mode (Windows High Contrast).

forced-color-adjust-auto

Default - browser menyesuaikan warna otomatis di high contrast mode

forced-color-adjust-none

Preserve warna asli (gunakan dengan hati-hati!)

‚ö†Ô∏è Perhatian: Hanya gunakan forced-color-adjust-none jika benar-benar diperlukan, seperti untuk preserving brand colors atau syntax highlighting.

4 Focus States

Pastikan semua elemen interaktif memiliki focus state yang jelas untuk keyboard navigation.

focus:ring-2 - Ring width
focus:ring-{color} - Ring color
focus:ring-offset-2 - Gap dari element
focus:ring-offset-{color} - Offset color
focus:outline-none - Remove default
focus:outline-2 - Custom outline

5 Focus Visible

Hanya tampilkan focus ring saat keyboard navigation, bukan mouse click.

<button class="focus-visible:ring-2 focus-visible:ring-blue-400
               focus-visible:ring-offset-2">
  Click atau Tab
</button>

üí° Difference: focus: muncul di klik & keyboard, sedangkan focus-visible: hanya muncul di keyboard navigation.

6 Color Contrast

WCAG AA membutuhkan contrast ratio minimal 4.5:1 untuk teks normal.

‚úÖ Baik: text-white + bg-gray-800

Contrast ratio tinggi, mudah dibaca

‚ùå Buruk: text-gray-600

Contrast terlalu rendah

Rekomendasi Colors untuk Dark Mode:

text-white - Untuk heading utama
text-gray-100 - Body text
text-gray-300 - Secondary text ‚úÖ
text-gray-400 - Muted text (minimal)
text-gray-500 - ‚ö†Ô∏è Borderline!

7 Motion Preferences

Hormati preferensi user yang tidak ingin animasi (vestibular disorders).

motion-safe:animate-*

Animasi hanya jika user mengizinkan motion

motion-reduce:transition-none

Matikan transisi untuk reduced motion

<!-- Bounce hanya jika motion allowed -->
<div class="motion-safe:animate-bounce">...</div>

<!-- Disable transition untuk reduced motion -->
<button class="transition-all motion-reduce:transition-none">
  Button
</button>

8 Semantic HTML

‚úÖ Gunakan:

  • <button> untuk clickable actions
  • <a href> untuk navigation
  • <nav> untuk navigation wrapper
  • <main> untuk main content
  • <header> / <footer>
  • <article> / <section>
  • <h1>-<h6> dengan urutan benar

‚ùå Hindari:

  • <div onclick> sebagai button
  • <span> sebagai heading
  • <div role="link"> (pakai <a>)
  • Skip heading levels (h1 ‚Üí h3)
  • Multiple <h1> per page
